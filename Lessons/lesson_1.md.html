Atom Basics

Honestly there isn't much to do here, Atom runs pretty darn well out of the box, and takes little configuring. I would explore the main settings a bit, maybe look at the themes to find one that is easy on your eyes, and fiddle with word wrap, which you can search for. I usually like word wrap because it reduces the amount of side scrolling I do, but other people like how they can see the indent structure of their file better. Two great resources to learn all the ins and outs of Atom are provided below.

  • Atom Flight Manual
  • Atom Docs

There are two key features I am going to focus on that are pretty much core to the whole process and will be found in all good editors, as well as a third that is a little unique to Atom that I thoroughly enjoy.

The first I also go over in the VS Code section, and I wrote that first, so if you're really interested in why I like it go check that out in the section just after this for a little more info. The tool here is called 'Tree View' and it can be toggled from the View menu or with Ctrl+\. This should bring up a panel on the right, but to get the most out of it we have one more step. Under the File menu select Open Folder... and browse to where this file is currently stored.

The next important tool is something that comes built right into Atom. This is the Markdown preview. As you might have noticed this file has some random #s and some other odd symbols scattered throughout it. This is a Markdown file. In fact in the browser you just opened on the left, you should see something like lesson_1.md and if you click on the file Atom will bring up an editor and should provide some nice color highlighting of all those symbols. This is because Atom excels at handling Markdown and already has a suite of tools to work with it built in. The first of these tools we have seen is the highlighting. The second that we will use is preview I mentioned. To open this type Ctrl+Shift+M on Windows and Cmd+Shift+M on Mac. This should also trigger another nice feature of Atom, the split view editor. On the right you should now see a file that has all the same words as the one on the right, but non of the odd symbols and has some nice formatting, like font size and links and some code snippets. This is what Markdown looks like when it is interpreted and rendered by something that knows how to. The basic text editor you were using before to read this file didn't know how to display it, at least not right off the bat.

The final feature really leads us into a different section, but I will mention it now and we will sort of come back to it in a bit. Under the same View menu you will see a Toggle GitHub Tab option, which can also be triggered with Ctrl+Shift+8 or by the small button in the lower right. Open this and hopefully you will be greeted by a login button, head on to the next steps and we will come back to this, but basically the system we will use to host and track all our files integrates directly into the side of Atom and that is wonderful.

VS Code Basics

Now that you have those installed I'll run you through the super basics of VS Code and it's interface. A lot of the customization potential takes a lot of getting used to, so check out the VS Code tutorials and such.

  • VS Code Docs

There are a few things I am going to focus on. The first and arguably most important is the file browser on the left. To show this you might need to click the icon that looks like two stacked pages, should be first in the column. This file browser is incredibly powerful and you'll learn to love it. Most text editors made for coding have something like this.

The first thing we are going to do with this new tool is navigate to the folder of your choice. Back in VS Code click the blue 'Open Folder' button in the left hand column and find the folder where this file is being stored. You'll see a folder tree open on the side and select this file within it which will open it in an editor window. This leads us into the next important feature.

The other important part of VS Code is the command console. This can be accessed by typing 'ctrl+shift+p'. An input box should appear at the top of the screen which you can type in. There are some recommend commands and recent ones as well, this can be a really fun thing to explore because it is very powerful. The command we are looking for can be found by typing Markdown. One of the options that should appear is 'Markdown: Open Preview'. As long as you still have this file open in a VS Code editor it should open a new tab. Clicking on this tab will show you a very different looking file. This is what the formatting language called Markdown looks like when rendered. Now if you press the split view button in the upper right your editor window will split down the middle and you can drag the tab of the Markdown preview to the other half of the screen, exposing the raw file on the left. You can begin to explore this file to see what raw text causes what rendered output. For info I use GitHub's Markdown cheat sheet all the time.

  • Markdown Cheat Sheet

Explore Markdown for a bit, but let's put it aside for now and get ready to incorporate Git into our workflow.


Git Client

Git is one of many version control softwares out there, and most likely the widest used. It is incredibly versatile and powerful, but also keeps things simple enough for the average user to be able to get a lot out of it. Git will allow us to keep track of the changes we make over time, and if things go south will let us easily recover older versions of our work.

GitHub is in essence a web interface for Git, which a bunch of other features many of which we will use and get to know well, but our time with Git starts with finding a way we can easily interact with it on our computers.

To do this we need to download a tool to access Gut and GitHub through. This will work closely with VS Code for keeping track of edits to our work. For this we will use one of the lovely GUI (Graphical User Interfaces) tools that are out there, because coding doesn't have to be green on black and only in a terminal. GUIs are great, and I don't care if a bunch of older coders glare at me about back in their day they had to make the GUIs. I thank them for this and then will promptly download GitKraken. If you have another Git client you already know about and prefer go ahead and download that. The reason I recommend GitKraken, even though I know it has some problems, it has a very simple user interface, and will work perfectly for our uses. I personally like SourceTree as a more power user option, but I use GitKraken a lot as well.

Make Your First Repo

Once you are logged in click the '+' icon in the upper right corner, and then select 'New Repository'. Fill out the text fields, which should be something like:

  • Repository Name
  • Description
  • Initialize
  • Git ignore and license can be ignored for now

When you are satisfied with the various fields click the 'Create Repository' button. Repositories are cheap and can be easily deleted, so we aren't necessarily going to use this specific one for much beyond this initial tutorial. For now we have a bit to do with it, but to do that we need to be able to access it on our desktop, where VS Code can interact with its file structure. This is where we will bring everything we just learned together.

GitHub and GitKraken Linking

Launch GitKraken and under the file menu (Windows) or the main menu item (Mac) select preferences. Then on the left select 'Authentication' and then the GitHub tab. I am fairly certain you just click a single button and it launches a browser and takes you to GutHub where there will be a button to allow GitKraken access. This should switch you back to GitKraken. Now back out to the main page and click the folder icon in the top left. This will open a page with a column of words down the left side. Select the 'Clone' option. You should then see a 'GitHub' tab and upon clicking on it the repository you made in the previous step should be an option. Next to the 'Where to Clone' box click 'Browse' and pick a spot for the repository. Click clone and GitKraken should ask you if you would like to create an initial commit, which you can just say yes to, we will learn more about commits later. After that it should dump you back out to the main screen but a few things will be different. You see a line in the middle that says 'Initial Commit' and the upper left corner should reflect the name of repository you are in.
